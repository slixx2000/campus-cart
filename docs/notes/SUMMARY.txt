Q1: Can users download the app right now?
A: Not yet. The download page is live, but it links to Expo EAS where the APK
   needs to be built first. You need to run: eas build --platform android

Q2: Do I need to modify any code after building the APK?
A: No, the page is fully functional as-is. Optionally, you can self-host the
   APK in public/downloads/ and update the link in page.tsx.

Q3: Will Android users see a better experience?
A: Yes! The download page auto-detects Android and highlights the download
   button prominently with your primary brand color.

Q4: What about iOS users?
A: iOS shows as "Coming Soon". The same Expo process works for iOS builds.
   Plan for that after Android is successful.

Q5: Can I deploy just the web changes now?
A: Yes! The download page is complete and fully functional even without an APK.
   Users can see it, but would go to Expo EAS to download.

Q6: How long does an APK build take?
A: ~10-15 minutes using Expo EAS cloud build. No local setup needed!

Q7: Do I need Docker/Android SDK installed?
A: Not with Expo EAS! That's why it's recommended. Everything runs on Expo's
   servers. If you want local builds, see ANDROID_BUILD_GUIDE.md.

Q8: Can I host the APK on my own server after building?
A: Yes! Copy the APK to public/downloads/ and update the button link in
   src/app/downloads/page.tsx.
